

\# robtrace\_show



本系统为毕业设计项目，是一个基于 C++ 语言开发的 \*\*UR5 机械臂实时数据采集与轨迹可视化系统\*\*。项目基于 Visual Studio + Qt + VTK 架构开发，实现了对机械臂运行状态的实时监控、数据采集（采用自定义数据结构类型）以及三维轨迹的可视化渲染。



\---



\## 💻 运行环境与依赖



本项目环境配置较为复杂，建议严格按照以下版本进行配置。\*\*注意：本项目目前仅支持 Windows 11 (x86 架构) 的 Release 模式编译。\*\*



| 依赖项 | 推荐版本/架构 | 备注 |

| :--- | :--- | :--- |

| \*\*操作系统\*\* | Windows 11 (x86) | 核心编译目标架构为 \*\*x86\*\* |

| \*\*IDE\*\* | Visual Studio 2022 | 需安装 `Qt Visual Studio Tools` 插件 |

| \*\*Qt\*\* | Qt 5.12 | 界面与多线程数据采集 |

| \*\*VTK\*\* | VTK 9.4.2 | 用于三维轨迹的实时渲染与可视化 |

| \*\*UR RTDE\*\* | 最新版 (基于 Boost 编译) | 优傲机器人实时数据传输接口 |

| \*\*Boost\*\* | Boost 1.86.0 (x86 / Release) | 用于支持 RTDE 库的编译与运行 |



> ⚠️ \*\*重要提示（避坑指南）\*\*：

> 由于第三方库依赖和兼容性问题，\*\*请务必切换至 `Release | x86` 模式进行编译\*\*。在 `Debug` 模式下编译会遇到无法解决的链接或运行时错误。



\---



\## 💾 数据集说明



\* \*\*数据格式\*\*：机械臂实际采集的运动数据，采用自定义的数据结构类型进行序列化存储。



\## 🚀 环境配置与运行指南



由于本项目依赖较多，环境配置可能需要花费较多时间，请按以下步骤配置你的 Visual Studio 工程：



\### 1. 配置 Qt 插件

1\. 用 VS 2022 打开 `robtrace\_show.sln`。

2\. 点击顶部菜单栏：`扩展 (Extensions)` -> `Qt VS Tools` -> `Qt Project Settings`。

3\. 在 `Version` 下拉菜单中，确保选择了你本地配置好的 \*\*Qt 5.12 (x86)\*\* 版本。



\### 2. 重新指定第三方库路径 (VTK / RTDE / Boost)

因每台电脑的库安装路径不同，请在编译前修改项目属性：

1\. 在右侧“解决方案资源管理器”中，右键点击 `robtrace` 项目 -> 选择 \*\*属性\*\*。

2\. 将左上角的 \*\*配置\*\* 切换为 `Release`，\*\*平台\*\* 切换为 `Win32` (即 x86)。

3\. \*\*C/C++ -> 常规 -> 附加包含目录\*\*：

&#x20;  \* 添加你本地的 `VTK 9.4.2` 头文件路径

&#x20;  \* 添加 `RTDE` 头文件路径

&#x20;  \* 添加 `Boost 1.86.0` 的根目录

4\. \*\*链接器 -> 常规 -> 附加库目录\*\*：

&#x20;  \* 添加你本地对应的 `VTK/lib` (x86 Release版)

&#x20;  \* 添加 `RTDE/lib` (x86 Release版)

&#x20;  \* 添加 `Boost/stage/lib` (x86 编译产物)



\### 3. 编译与一键运行

1\. 确认 VS 顶部工具栏的编译选项为：\*\*`Release`\*\* 和 \*\*`x86`\*\*。

2\. 按下 `F5` 或点击 \*\*本地Windows调试器\*\* 即可成功编译并运行系统。
